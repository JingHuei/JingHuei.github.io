［AC 黑客松］開始做自己的 Side-Project

發表於2022-04-02|更新於2022-04-02|學習心得
|閱讀量:

前言

從 AC 的課第一次聽到 Side-Project 之後，網頁技術還學不深的我並沒有太多想法，後來想轉職或是想申請像 AppWorks School 這類培訓營，看到條件都要求作品集，就萌生想製作屬於自己的 Side-Project 的念頭，但我印象中的 Side-Project 好像都要很厲害才拿得出來，我就開始先從日常生活中發想，有什麼主題可以製作，覺得先有想法再來一步一步實作完成。

因為不論是找工作或是申請培訓營都需要履歷表，也剛好在練習 BootStrap ，就先用 BootStrap 刻出一個屬於自己的一頁式網站履歷，完成這個網頁非常有成就感～

今後也多了屬於自己的排版的網頁履歷表可以用，我覺得超棒！

後來陸續又想了好多點子，但每次要實作的時候都因為腦袋思緒太過混亂、網頁技術程度不足不懂得怎麼做才能達到目標、、、等好多原因放棄‧（簡單來說就是自己懶又容易放棄） ╮(╯∀╰)╭

不過一直都還是有在嘗試拿別人做過的專案來練習，某一天做到 todo-list 的時候，突然靈機一動想到，這樣的東西好像可以拿來做一點延伸！

於是開啟我的小小 Side-Project 專案目標　( • ̀ω•́ )

埋頭苦幹

做好了一個 todo-list 後，剛好在看自己的戶頭餘額可憐的剩無所幾，想著我不是有存錢嗎！都存去哪了？　( ´•̥̥̥ω•̥̥̥` )

立刻想到延伸專案的雛型，並開始規劃！

先在紙上大概規劃，畫了想要的排版、要有什麼資料呈現、要有什麼功能，然後開始用原本的 todo-list 專案去延伸做實作。

我想做的延伸專案 - 資產一目瞭然的管理系統

首先完成最最主要的功能，就是上方新增的資料可以傳到下方的清單，但功能因為碰到了以下困難還沒完整實作出來。

  • todo-list 的只有欄位一個，也就是說新增後要傳入到清單裡的值只有一個，延伸後我想傳入到清單裡的資料欄位有六個，我不知道要怎麼讓六個值順利依照欄位傳到下方清單的格子裡，並且讓資料每新增一列都有該列獨一無二的一組 ID 。

  • todo-list 的輸入框 type 是文字，延伸後新增了日期及下拉式選單，我不知道這兩個的值要怎麼抓取和新增到清單裡。

光是最基本的功能就讓我非常頭大，自己上網找解決方法，已經能讓下拉式清單成功抓取值並傳到下方清單裡，但光是卡在第一點就讓我非常不知所措，只好不斷查資料研究和嘗試解決。

專案擺著，我的點子卻沒停，一直不斷的抱持期待，想在這個專案增加好多功能，雖然現階段不會做，但總想著應該能慢慢研究出方法吧。

發現好機會

以前看到 AC 黑客松的活動資訊都不敢報名，因為我一點想法都沒有啊！而且還要分享自己的想法和進度，我．．．很害羞，一定不行啦．．．　(つд⊂)

但專案卡關的期間再次看到黑客松活動，居然湧起了報名的熱情，事不宜遲馬上報名！分享什麼的也不害怕了！

想像中會參加這個活動的人感覺都很厲害，對活動有各種疑問像是「我這麼菜真的能參加嗎？」、「我的專案還沒做好，參加活動一定要當天完成嗎？」、「如果問問題大家會不會覺得我的問題很好笑？」就算趁著衝動和突然燃起的熱情報名了，在報名後還是抱持著這類的自我懷疑。

好險這些問題都在社群裡助教的回應給了一種安心的力量

關鍵字 > 做不完沒關係 XD

也因為助教說可以找他聊聊，我感覺很像找到救星，覺得是大好機會！

很想在活動當天抓著助教猛問，哈哈

活動開始

我的黑客松初體驗

從來沒參加過這類活動的我，連假第一天抱著學習的熱情起床了　(揉眼還沒醒

進入 AC Gather.town 發現好像小遊戲好有趣，活動開始前助教講解了幾點關於做 Side-Project 的事項，我覺得有小收穫。

  • 每個人想做 Side-Project 的初衷不一定一樣

    有的人想透過專案凸顯自己、有的人想透過作專案練習平常不會用到的技術、有的人想自己做專案來解決日常生活中的問題。我就是最後者想要自己做一個專案來解決自己的理財問題。

  • 找專案靈感的方法

    如果沒有想法的話，可以嘗試複製別人網站很酷炫的功能、在網路上找專案參考、想一下日常生活中有沒有頻繁遇到什麼問題想透過程式有效率的解決。

    像我就覺得每天要想午餐要吃什麼，蠻選擇障礙的，之後想自己做個決定午餐吃什麼的小程式幫助自己做決定 XD

  • 缺乏做 Side-Project 的意志力的話，可以找人互相監督有沒有達到目標。

  • 有些 Side-Project 是初學者目前無法完成的，但現階段無法完成沒關係，隨著未來學到的技能變多，說不定累積到一定的程度之後就能做得出來～

  • 專案的範圍太大的話會無法完成。

  • 做一個 Side-Project 比較像是從完成簡單的小專案開始，再慢慢進步到能做出越來越大越好的專案，而不是從完成一個專案的小區塊開始完成到大專案。

    看到這張圖真的修正我既有的想法，因為我以為的做專案真的就像上圖的模式，結果原來是錯誤的，做一個 Side-Project 先不要管規模，以目前自己的能力完成才是最重要的！

活動開始隨著助教的講解及帶領，我逐漸放下擔心做不好和害羞的心情，融入社群和大家有點小小互動，

迫不及待的想問助教問題，但止步了，因為思緒混亂的我根本還沒釐清自己的問題，在 AC 有學到問問題之前，先釐清自己的問題點，因為自己都不知道問題在哪裡，別人也不知道能夠如何幫助你，自己也會不知道怎麼提問比較好。

因此我像活動前分享的同學一樣，先做了簡單的 PPT 為自己講解一遍我的專案在做什麼、想要有什麼功能、我遇到什麼問題，幫助我釐清問題的同時也能更好讓助教知道我的問題在哪。

向助教提問完之後得到的回覆是我把專案製作想得太困難了，因為我目前的程度在學期 2-1 已經完成，剛開始預讀學期 2-2 的階段，因此助教建議我先以完成 HTML/CSS 切版為主，至於清單內紀錄資產的項目可以先以自己加入 HTML 來代替，等後面學到比較多的時候再來完成想完成的功能。

我只能說得到這個建議之後整個人超級放心的，像是告訴我不要緊逼自己要急著去完成還不太熟的部分，先把自己會的練熟。

於是在剩下的時間裡，我把心思放在如何優化我的畫面視覺體驗，原本的畫面屬於純手刻，後來回頭省思自己的版面還有哪裡調整一下會更好，想到表格單數列可以增加底色，並讓使用者滑鼠停留該列的顏色加重，所以引入 BootStrap 來完成這項功能，也將整個版面套用 BootStrap 效果，確實有增加一點互動性提升網頁體驗。

第一次黑客松終點

今天的黑客松初體驗感覺很好，聽同學的分享得知好多日常生活的瑣事原來可以發想成專案，還有一些在 Gather.town 真的蠻有趣，最重要的是輪到我分享的時候，我竟然可以自信的透過語音分享我的專案想法，雖然還是有點緊張 (抖)，但至少我沒有逃避有完成分享～　d(`･∀･)b

參加完整場活動釐清很多自己的問題點，調整了做專案的進度及該抱持的心態，也讓我的思緒清晰很多。

完成了符合現階段程度的小專案提升很多自信心及成就感，對接下來的要完成的目標也抱持著很多的期待和動力！

活動結束前的大合照，真的好歡樂有趣～
